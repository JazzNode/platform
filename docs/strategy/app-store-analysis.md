# JazzNode App Store 策略分析

> 分析日期：2026-03-21

## 現況摘要

JazzNode 是一個「爵士樂的 IMDb」— 全球爵士活動探索與社群平台，目前為純 Web 應用（Next.js 16 + Supabase），採用 Freemium 訂閱制（Tier 0-3），支援 6 種語言，覆蓋亞洲多個城市。目前沒有原生 App、沒有 PWA、也沒有 App Store 相關規劃。

## 建議：現階段不應推出原生 App，應先採用 PWA 策略

### 1. 成本與資源不對稱

| 項目 | 原生 App | PWA |
|------|---------|-----|
| 開發成本 | 高（需 iOS + Android 雙平台） | 低（基於現有 Next.js） |
| 維護成本 | 持續雙倍維護 | 與 Web 同步 |
| 上架審核 | Apple 審核嚴格、耗時 | 無需審核 |
| 更新部署 | 需用戶手動更新 | 即時部署 |

JazzNode 目前仍在 MVP 階段，功能快速迭代中。此階段最大的優勢是 Web 的即時部署能力，原生 App 的審核週期會嚴重拖慢迭代速度。

### 2. 商業模式尚未驗證

- Tier 1-3 的付費訂閱尚未正式上線
- 用戶基數和付費轉換率都還是未知數
- 在商業模式驗證前投入原生 App 開發，是過早的資源配置

### 3. Apple 30% 抽成問題

如果在 App Store 內處理訂閱付款，Apple 會抽取 15-30% 的收入。對於 JazzNode 這種創作者工具型平台（藝術家/場館付費解鎖進階功能），這個抽成直接侵蝕利潤。Web 端付款可以完全避開這個問題。

### 4. 核心使用場景適合 Web

- **活動探索 / 瀏覽**：低頻、資訊導向 → Web 搜尋 + SEO 更有效
- **藝術家/場館管理後台**：表單操作、數據分析 → 桌面 Web 體驗更佳
- **Magazine 閱讀**：長內容消費 → Web 即可
- JSON-LD 結構化資料規劃也說明 SEO 是重要獲客渠道，App Store 生態中無法實現

### 5. Push Notification 不需要原生 App

Roadmap 中的 Push Notifications 是唯一需要「接近原生」能力的功能，但 PWA 已支援 Web Push（iOS 16.4+ 也支援）。

## 建議的分階段策略

```
Phase 1（現在）  → 加入 PWA 支持（manifest.json + Service Worker）
                    成本極低，立即獲得「加到主畫面」能力

Phase 2（驗證期）→ 驗證付費模式，觀察 DAU/MAU、付費轉換率
                    透過 Web Push 實現通知功能

Phase 3（成長期）→ 當以下條件同時滿足時，再考慮原生 App：
                    ✅ 月活用戶 > 一定門檻（如 50K+）
                    ✅ 付費用戶佔比穩定
                    ✅ 用戶明確要求原生體驗
                    ✅ 有需要深度原生整合的功能（如 GPS 即時推播附近演出）
```

## 例外情況：何時應該考慮 App

- **市場競爭壓力**：如果競品已有高品質 App 且成為獲客壁壘
- **投資人/合作方要求**：部分 B2B 場景（場館合作）可能期望有 App
- **特定功能需求**：如即時位置推播、離線地圖、深度日曆整合等

## 結論

不建議現階段推出 App Store 應用。JazzNode 當前應專注於 Web 端的功能完善與商業模式驗證，以 PWA 作為過渡方案。在用戶基數和營收模式成熟後，再評估原生 App 的 ROI。過早投入 App 開發不僅浪費有限資源，還會因 Apple 抽成削弱本就未經驗證的收入模型。
