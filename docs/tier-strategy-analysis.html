JazzNode 內部文件 · 2026 年 3 月 · 基於資料庫架構與平台現況分析

一、核心命題：「藝人頁面即 EPK」策略評估

你的直覺是：與其花力氣做 EPK 自訂 PDF 下載，不如把藝人頁面本身做到像數位 EPK 一樣漂亮、資訊完整、讓人忍不住分享。

結論：完全正確。這是更好的策略。

PDF EPK 是一個靜態快照，而藝人頁面是一個活的、即時的、可互動的數位名片。從產品策略和開發效率兩個角度，把藝人頁面做成「最好的 EPK」遠優於開發自訂 PDF。

✅ 「藝人頁面即 EPK」的優勢

  • 頁面永遠是最新的，PDF 下載即過時
  • 頁面有互動性：播放音樂、看影片、一鍵聯繫
  • 自帶SEO，能被 Google 搜到
  • 可追蹤瀏覽數據（誰看了、從哪來）
  • 開發一次，全平台受益
  • 自然導流回平台，增加用戶量
  • 行業趨勢：Spotify for Artists、Bandcamp 都是頁面而非 PDF

⚠️ PDF EPK 的問題

  • 自訂 PDF 開發成本極高（@react-pdf 限制多）
  • PDF 無法嵌入音訊/影片
  • 字型、排版在 PDF 中很難完美
  • 下載後即脫離平台，無法追蹤
  • 「品牌化 PDF」需求因人而異，做不完
  • 多語言 PDF 渲染是噩夢（CJK 字型問題）

建議做法：保留一個簡單的 PDF 下載（現有的 basic 版本即可，做為離線備用），但把所有的產品力量投入「讓藝人頁面成為業界最漂亮的藝人展示頁」。PDF 不需要三階段差異，一個版本就夠了。

二、重新思考等級制度

現狀問題

目前規劃 4 個等級（Free / Claimed / Premium / Elite），但如果 EPK 不再是核心差異化賣點，Premium 和 Elite 的區隔就變得很模糊：

原本的核心差異拿掉 EPK 自訂後
Premium: EPK 完整版 + 分析❌ EPK 完整版不做了
Elite: 品牌化 PDF + 自訂域名 + 主題❌ 品牌化 PDF 不做了，自訂域名/主題開發成本極高

⚠️ 如果拿掉 EPK 差異化，Elite 幾乎沒有能快速上線的獨佔功能。自訂域名和自訂主題都是重度開發項目，短期內上不了線。

建議：簡化為三等級制

Free (0) → Claimed (1, 免費) → Pro (2, 付費)

先做一個付費等級，把 Premium 和 Elite 的精華功能合併。等用戶量和需求明確後，再考慮拆分更高等級。

為什麼？ 台灣 jazz 藝人是一個小眾市場（估計活躍藝人 200-500 人）。在這個規模下，兩個付費等級會導致每級的轉化人數太少，不如集中火力把一個付費等級做到有感。

三、基於現有資料庫的新功能機會

以下功能都是現有資料已支援、只需要寫前端 + API 就能上線的項目，不需要大幅改動 DB schema：

🔥 1. 進階分析儀表板（Advanced Analytics）

整合 artist_page_views、follows、gsc_search_analytics 三個表，呈現：

  • 流量趨勢：每日/週/月頁面瀏覽量折線圖
  • 粉絲成長：新增/流失粉絲趨勢
  • 搜尋關鍵字：哪些搜尋詞帶來流量（直接用 GSC 資料！）
  • 裝置分佈：桌面 vs 手機 vs 平板（GSC 已有 device 欄位）
  • 國家來源：流量來自哪些國家（GSC 已有 country 欄位）
📊 資料來源：artist_page_views + gsc_search_analytics + follows — 資料全部已在 DB 中

開發量：中 · 主要是前端圖表 + 聚合查詢 API

🔥 2. 粉絲 CRM（Fan Relationship Management）

整合 follows、conversations、messages、broadcast_deliveries，呈現：

  • 粉絲列表：所有追蹤者的 profile 資訊
  • 互動度排名：誰最常看你的頁面、誰開啟你的廣播
  • 訊息狀態：未讀訊息、對話時間軸
  • 通知偏好分佈：多少粉絲設定 All / Important / None
📊 資料來源：follows(含 notification_level) + conversations + broadcast_deliveries(含 read_at) — 資料全部已在 DB 中

開發量：中 · 列表 + 排序 + 聚合

🔥 3. SEO 洞察面板

你們已經在同步 Google Search Console 資料（gsc_search_analytics 表，每日 cron 同步），這是一個極度被低估的金礦。

  • 哪些搜尋詞找到你：「Sarah Chen jazz piano Taipei」這類關鍵字
  • Google 排名追蹤：你在哪些搜尋詞排第幾
  • 曝光 vs 點擊：哪些詞曝光高但點擊低（可優化標題）
  • 趨勢比較：這週 vs 上週的搜尋表現
📊 資料來源：gsc_search_analytics(含 query, clicks, impressions, ctr, position) — 資料已在 DB 中，每日同步

開發量：低 · 資料已有，只需前端呈現

🔥 4. AI 多語言翻譯

藝人頁面已支援 6 種語言（en/zh/ja/ko/th/id），但大多數藝人只填了 1-2 種。提供一鍵 AI 翻譯：

  • 簡介翻譯：寫了中文 bio，自動翻成英/日/韓/泰/印尼文
  • 教學描述翻譯：teaching_description 的 6 語言版本
  • 接案描述翻譯：hire_description 的 6 語言版本
  • 用 AI Gateway 即可實現，成本極低
📊 資料來源：artists 表已有 bio_en/zh/ja/ko/th/id + teaching_description_*/hire_description_* 欄位 — 欄位全部已建好

開發量：低 · AI Gateway + 6 語言欄位寫入

5. 雜誌專題推薦優先

你們已有 magazine_articles 表（含 linked_artist_ids），可以對 Pro 藝人提供：

  • JazzNode 編輯團隊優先考慮撰寫 Pro 藝人專題
  • 藝人頁面顯示「曾被 JazzNode Magazine 報導」標記
  • 雜誌文章連結直接嵌入藝人頁面
📊 資料來源：magazine_articles.linked_artist_ids — 關聯已建好

開發量：低 · 主要是營運策略 + 頁面展示

6. 接案收件匣強化

booking_inquiries 表已存在，但目前只是基礎表單。可強化為：

  • 結構化收件匣：依 event_type、budget_range 篩選
  • 快速回覆模板：「感謝您的邀請，我目前的檔期是...」
  • 狀態追蹤：新增 status 欄位（pending / replied / accepted / declined）
  • 歷史統計：月/季收到多少詢問、接案率
📊 資料來源：booking_inquiries(含 event_type, budget_range, event_date) — 表已建好，需加 status 欄位

開發量：中 · 加欄位 + UI + 模板系統

7. 合作網絡分析

lineups 表記錄了所有藝人與演出的關聯，可以算出：

  • 最常合作的藝人：你跟誰一起演出最多次
  • 合作場地分佈：你最常在哪些場地演出
  • 角色分析：bandleader vs sideman 的比例
  • 合作建議：基於共同場地/樂器推薦可能的合作對象
📊 資料來源：lineups(含 role, instrument_list) + events(含 venue_id) — 資料全部已在 DB 中

開發量：中 · 聚合查詢 + 視覺化

四、付費等級是否值那個錢？

讓我們誠實地問：一個台灣 jazz 藝人，願意每月付 NT$299 買什麼？

關鍵洞察：最能打動藝人的是三件事：
1️⃣ 「讓更多人找到我」（SEO 洞察 + 優先排名 + 多語言翻譯）
2️⃣ 「讓我直接賺到錢」（接案收件匣 + 廣播推送）
3️⃣ 「讓我知道誰在看」（進階分析 + 粉絲 CRM）

這三個方向都有現成的資料庫支撐，不需要從零開始。

NT$299/月值不值？

把上述功能打包成一個 Pro 方案，NT$299/月（年繳 NT$2,990）：

  • ✅ AI 翻譯一次就省了外包翻譯費 NT$3,000+（6 種語言）
  • ✅ 接案收件匣帶來一場演出就回本十倍
  • ✅ SEO 洞察等同免費的 Google Analytics 顧問
  • ✅ 進階分析每週看到數字成長就有動力繼續經營

結論：NT$299/月以這些功能組合來說是合理的，特別是 AI 翻譯和接案收件匣有明確的 ROI。

五、執行建議

Phase 路線圖

1

Phase 1：打磨藝人頁面（現在 → 免費功能）

目標：讓藝人頁面成為「最好的數位 EPK」

  • 優化頁面視覺設計（Hero 區、排版、響應式）
  • 加入「分享」功能（OG image 自動產生，社群分享按鈕）
  • 完善 Claimed 免費功能（基礎分析、收件匣）
  • 簡化 EPK PDF 為「一鍵下載基本版」
  • 推動更多藝人認領
2

Phase 2：上線 Pro 付費等級

目標：用最少開發量上線最有感的付費功能

  • 🔥 AI 多語言翻譯（低開發量、高價值感）← 殺手功能
  • 🔥 SEO 洞察面板（低開發量、資料已有）
  • 🔥 進階分析儀表板（中開發量、資料已有）
  • 可接案標示 + 接案收件匣強化
  • 精選演出牆 + 無限器材 + 無限廣播
  • Stripe 支付串接
3

Phase 3：深化付費價值

目標：根據付費用戶回饋，增加更多差異化功能

  • 粉絲 CRM（互動度排名、活躍粉絲辨識）
  • 合作網絡分析
  • 演後回顧 + 每週摘要信
  • 雜誌專題推薦（營運配合）
  • 若需求出現：考慮拆分 Elite 等級

最終建議

1. 藝人頁面即 EPK — 把所有產品力量投入讓藝人頁面成為最好的數位展示，PDF 只保留基本版

2. 簡化為三等級 — Free / Claimed（免費）/ Pro（付費 NT$299/月），不做 Elite

3. Pro 的殺手功能 — AI 多語言翻譯 + SEO 洞察 + 進階分析 + 接案收件匣

4. 利用現有資料 — GSC 搜尋數據、follows、page_views、booking_inquiries 都已在 DB 中，開發成本低

5. 先證明價值再擴張 — 等有 30+ 付費用戶且有明確需求時，再考慮拆分 Elite 等級
